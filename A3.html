Description of Malware

Malware is an umbrella term for any type of “malicious software” that’s designed to infiltrate your device without your knowledge, cause damage or disruption to your system, or steal data. Adware, spyware, viruses, botnets, trojans, worms, rootkits, and ransomware all fall under the definition of malware.

Souce: https://www.avast.com/c-malware
2.

Denial-of-Service (DoS) Attacks

Image source:https://www.thesslstore.com/blog/wp-content/uploads/2019/10/what-is-ddos-botmaster.png

Description of Denial-of-Service (DoS) Attacks

A denial-of-service (DoS) attack is a type of cyber attack in which a malicious actor aims to render a computer or other device unavailable to its intended users by interrupting the device's normal functioning. DoS attacks typically function by overwhelming or flooding a targeted machine with requests until normal traffic is unable to be processed, resulting in denial-of-service to addition users. A DoS attack is characterized by using a single computer to launch the attack.

Souce: https://www.cloudflare.com/en-gb/learning/ddos/glossary/denial-of-service/

Description of Phishing

Phishing is a cybercrime in which a target or targets are contacted by email, telephone or text message by someone posing as a legitimate institution to lure individuals into providing sensitive data such as personally identifiable information, banking and credit card details, and passwords.

Spoofing

Image source:https://lh3.googleusercontent.com/-_4ah0CSvzZM/ZAf6ZhEUDGI/AAAAAAAAB0A/AxtkpYKbJRUmE9nrgTvxaBoIIHkVZwrdACNcBGAsYHQ/s16000/IMG_ORG_1678244442259.jpeg

Description of Spoofing

Spoofing is a type of scam in which a criminal disguises an email address, display name, phone number, text message, or website URL to convince a target that they are interacting with a known, trusted source. Spoofing often involves changing just one letter, number, or symbol of the communication so that it looks valid at a quick glance. For example, you could receive an email that appears to be from Netflix using the fake domain name “netffix.com.”

Souce: https://www.investopedia.com/terms/s/spoofing.asp

Identity-based attacks occur when cybercriminals target your computer system, network, or account to retrieve your personal information for illegal or malicious activities. Also known as impersonation attacks, threat actors leverage them to collect sensitive data, steal money, and ruin the target's reputation.

Code Injection Attacks

Image source:https://www.cobalt.io/hubfs/Imported_Blog_Media/A_Pentester___s_Guide_to_Code_Injection-1.png

Description of Code Injection Attacks

Code injection is the term used to describe attacks that inject code into an application. That injected code is then interpreted by the application, changing the way a program executes. Code injection attacks typically exploit an application vulnerability that allows the processing of invalid data. This type of attack exploits poor handling of untrusted data, and these types of attacks are usually made possible due to a lack of proper input/output data validation. Attackers are able to introduce (or inject) code into a computer program with this type of vulnerability.

Code injection differs from command injection, where the goal is to hijack a vulnerable application in order to execute arbitrary commands on the host operating system. Command injection consists of leveraging existing code to execute commands, usually within the context of a shell.

Souce: https://www.contrastsecurity.com/glossary/code-injection#:~:text=Code%20injection%20is%20the%20term,the%20processing%20of%20invalid%20data.
